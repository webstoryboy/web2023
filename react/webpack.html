Webpack

Webpack은 JavaScript 애플리케이션을 위한 정적 모듈 번들러입니다.

웹 애플리케이션을 구성하는 다양한 모듈(JavaScript, CSS, 이미지 등)을 하나의 번들로 묶어줍니다.

웹 애플리케이션은 보통 여러 개의 JavaScript 파일과 종속성을 가지는데, 이러한 파일들을 하나의 번들로 묶으면 네트워크 로딩 속도를 향상시킬 수 있습니다. 또한, 웹 애플리케이션 개발 중에는 ES6 모듈 시스템을 사용하거나 다양한 프런트엔드 프레임워크나 라이브러리를 사용하는 경우가 많은데, 이를 효율적으로 관리하고 번들로 만들어주는 역할도 수행합니다.

Webpack은 다양한 로더(loader)와 플러그인(plugin)을 제공하여, CSS 파일, 이미지 파일 등과 같은 다른 종류의 파일도 모듈로 취급하고 처리할 수 있습니다. 또한, 코드 압축, 트랜스파일링, Hot Module Replacement(HMR) 등의 기능을 제공하여 개발 및 배포 단계에서도 유용하게 사용할 수 있습니다.

1.4 webpack.config.js 설정

npx webpack을 작성하면 파일이 번들링 됩니다.

const path = require('path');

module.exports = {
    mode: 'production', //production, development
    entry: {
        index: './src/index.js'
    },
    output: {
        path: path.resolve(__dirname, 'public'),
        filename: 'js/[name].js'
    }
}

1.6 HTML 번들러 설정

npm install html-webpack-plugin --save-dev
"devDependencies": {
    "html-webpack-plugin": "^5.5.1",
    "webpack": "^5.83.1",
    "webpack-cli": "^5.1.1",
    "webpack-dev-server": "^4.15.0"
}

const path = require('path');
const HtmlWebpackPlugin = require('html-webpack-plugin');

module.exports = {
    mode: 'production', //production, development
    entry: {
        index: './src/index.js',
    },
    output: {
        path: path.resolve(__dirname, 'public'),
        filename: 'js/[name].js',
    },
    plugins: [new HtmlWebpackPlugin({
        template: './src/index.html',   //번들 전
        filename: './index.html',    //번들 후 
        hash: true,
        showErrors: true
    })],
}
